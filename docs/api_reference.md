# ChairSleep API リファレンス

本プロジェクト（`apps/server`）で提供されている機能ごとの内部APIエンドポイントの一覧です。
すべてのAPIリクエストは `/api` がベースURLとなります。（例: `http://localhost:4000/api/...`）

---

## 認証 (Auth)
エンドポイントのプレフィックス: `/api/auth`

| メソッド | エンドポイント | 説明 | 認証 |
| :--- | :--- | :--- | :---: |
| `POST` | `/signup` | メールアドレスとパスワードによる新規ユーザー登録。成功時にCookieにJWTをセットします。 | 不要 |
| `POST` | `/login` | メールアドレスとパスワードによるログイン。成功時にCookieにJWTをセットします。 | 不要 |
| `GET` | `/line` | LINEログイン（OAuth）用のアカウント認証画面へのリダイレクトURL。 | 不要 |
| `GET` | `/line/callback` | LINEログイン完了後のコールバックエンドポイント。自動でユーザー作成・ログインを行いJWTをセットします。 | 不要 |
| `GET` | `/me` | 現在ログインしているユーザーの情報を取得します。 | **必須** |
| `POST` | `/logout` | ログアウト処理。CookieからJWTを削除します。 | 不要 |

---

## 投稿 (Posts)
エンドポイントのプレフィックス: `/api/posts`

| メソッド | エンドポイント | 説明 | 認証 |
| :--- | :--- | :--- | :---: |
| `GET` | `/` | 投稿の一覧を取得します。最新順に並びます。 | 不要 |
| `POST` | `/` | 新しい投稿（タイトル、内容など）を作成します。 | **必須** |
| `GET` | `/:id` | 特定のIDの投稿詳細を取得します。紐づいている登録アイテム（Items）やユーザー情報も併せて返却されます。 | 不要 |
| `PUT` | `/:id` | (*実装予定*) 特定のIDの投稿を更新します。 | **必須** |
| `DELETE` | `/:id` | 特定のIDの投稿を削除します。 | **必須** |

---

## アイテム・楽天API連携 (Items)
エンドポイントのプレフィックス: `/api/items`

| メソッド | エンドポイント | 説明 | 認証 |
| :--- | :--- | :--- | :---: |
| `GET` | `/search` | 外部の「楽天商品検索API」を利用し、クエリ（`?keyword=...`）に一致するアイテムを検索して返却します。 | **必須** |
| `POST` | `/` | ユーザーの投稿（Post）に対して、新しい紹介アイテムを登録します。 | **必須** |
| `DELETE` | `/:id` | 投稿に紐付いた特定のアイテムを削除します。 | **必須** |

---

## エラーレスポンスと認証

本APIでは、認証情報のやり取りに **HTTP Only Cookie** を使用しています。
そのため、フロントエンド（React/Vite）からフェッチを行う際は、自動的にCookieが送信されるためAuthorizationヘッダーの設定などは不要です。（※CSRF対策等が導入される場合は別設定が必要になることがあります）

認証が必要（**必須**）なエンドポイントに対して、未ログインや無効なCookieでリクエストを行った場合は、ミドルウェアによって以下のエラー・ステータスコードが返却されます。

```json
// HTTP Status: 401 Unauthorized
{
  "error": "Unauthorized"
}
```

## PrismaのBigInt対応について
`GET /api/posts` など、Prismaを介してデータベースからデータを取得する一部のエンドポイントにおいて、IDなどに `BigInt` が含まれるケースがあります。
これらはAPIレスポンスとして返却される直前に `serializeBigInt` ヘルパーを通るため、フロントエンド側には常に**文字列(String)**として安全にシリアライズされて届出されます。
